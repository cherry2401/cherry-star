## 🔴 Critical / Blocker Deploy (Phải sửa trước khi deploy)

### 1. Backend build fail — `tsc` compile error
- **File:** [package.json](file:///I:/Website/Auto-like/server/package.json), [tsconfig.json](file:///I:/Website/Auto-like/server/tsconfig.json)
- **Lỗi:** `TS4023: Exported variable 'db' has or is using name 'BetterSqlite3.Database' from external module but cannot be named`
- **Hậu quả:** `npm run build` → `tsc` fail → không tạo được `dist/` → `npm start` (production) không chạy được
- **Cách sửa (chọn 1 trong 3):**
  - Sửa cách export/annotate biến `db` trong `database/index.ts` (thêm type annotation tường minh)
  - Tắt `declaration: true` trong `tsconfig.json` nếu project không cần emit `.d.ts`
  - Thay đổi cách import/type để declaration generator xử lý được

### 2. Frontend API routing — không có `baseURL` cho production
- **File:** [api.ts](file:///I:/Website/Auto-like/src/services/api.ts), [vite.config.ts](file:///I:/Website/Auto-like/vite.config.ts)
- **Vấn đề:** `api.ts` tạo axios instance **không có baseURL** → hoạt động nhờ Vite dev proxy (`/api` → `localhost:3001`). Khi build production (`npm run build`), Vite proxy không tồn tại → toàn bộ API call sẽ **404**
- **Cách sửa:** Cần một trong hai:
  - Thêm `baseURL` dựa trên env variable (`VITE_API_URL`)
  - Hoặc deploy qua reverse proxy (Nginx/Caddy) route `/api` → backend

### 3. `NODE_ENV` + `CORS_ORIGIN` chưa cấu hình production
- **File:** [.env](file:///I:/Website/Auto-like/server/.env)
- **Vấn đề:** `NODE_ENV=development` → rate limiting lỏng (100 thay vì 15 req/15m cho auth), các production guard trong `config.ts` không kích hoạt. `CORS_ORIGIN` rỗng → production frontend bị block
- **Tính chất:** Misconfiguration risk, không phải lỗ hổng code tự thân. Nhưng là **blocker để deploy đúng cách**

## 🟡 High Warning (Nên sửa trước deploy)

### 1. Admin password mặc định `Admin@123456`
- **File:** [.env](file:///I:/Website/Auto-like/server/.env) dòng 17
- **Lưu ý:** Code `config.ts` **có enforce** admin password trên production mode. Vấn đề cốt lõi là nếu `NODE_ENV` sai → nhánh production không chạy. Root cause nằm ở mode/config, không phải auth logic sai
- **Cách sửa:** Đổi password mạnh hơn trong `.env` production

### 2. SQLite + migration ad-hoc
- **File:** [database/index.ts](file:///I:/Website/Auto-like/server/database/index.ts)
- **Vấn đề:** Migration chạy bằng `try/catch ALTER TABLE` — không có version tracking, không rollback. Phù hợp cho MVP/single-server, nhưng rủi ro khi scale hoặc có nhiều schema changes
- **Đánh giá:** Chấp nhận được cho giai đoạn hiện tại, nhưng cần lưu ý nếu project mở rộng

### 3. Thiếu unified error handling ở frontend
- **File:** [api.ts](file:///I:/Website/Auto-like/src/services/api.ts), [AuthContext.tsx](file:///I:/Website/Auto-like/src/contexts/AuthContext.tsx)
- **Vấn đề:** Chưa có response interceptor thống nhất xử lý 401 (auto logout), 500 (retry), network error. Mỗi component tự `catch` riêng
