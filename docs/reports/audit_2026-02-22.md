## 🔴 Critical Issues (Phải sửa ngay)

### 1. API Key bị lộ ra Frontend `.env`
- **File:** `.env` (root)
- **Chi tiết:** `VITE_API_KEY=bm93MTBkZGU3NjhlZWRkN2Jh...` — Bất kỳ biến nào bắt đầu bằng `VITE_` sẽ được Vite nhúng vào bundle JavaScript, ai cũng xem được qua DevTools.
- **Đời thường:** "API key của BaoStar đang bị lộ. Bất kỳ ai mở trang web đều có thể lấy key này và dùng dịch vụ BaoStar miễn phí bằng tài khoản của anh."
- **Cách sửa:**
  1. Xóa `VITE_API_KEY` khỏi `.env` gốc
  2. Đảm bảo API key chỉ tồn tại trong `server/.env` (đã có sẵn)
  3. Kiểm tra frontend code không còn gọi trực tiếp BaoStar API

### 2. JWT Secret quá yếu
- **File:** `server/.env` dòng 6
- **Chi tiết:** `JWT_SECRET=autolike_jwt_secret_change_in_production_2024` — Dễ đoán, nằm trong commit history.
- **Đời thường:** "Hiện tại secret dùng để ký token đăng nhập quá đơn giản. Hacker có thể giả mạo token admin, login vào bất kỳ tài khoản nào, nạp tiền tùy ý."
- **Cách sửa:**
  ```bash
  node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
  ```
  Copy kết quả vào `JWT_SECRET` trong `server/.env`

### 3. Mật khẩu Admin mặc định bị hardcode
- **File:** `server/.env` dòng 17, `server/config.ts` dòng 44
- **Chi tiết:** `ADMIN_PASSWORD=Admin@123456` — Fallback password nằm trong source code.
- **Đời thường:** "Ai đọc được source code đều biết mật khẩu admin mặc định. Kẻ tấn công chỉ cần thử `admin / Admin@123456` là có toàn quyền hệ thống."
- **Cách sửa:**
  1. Đổi mật khẩu admin ngay lập tức
  2. Sửa `config.ts` để **không** có giá trị fallback — buộc phải set qua `.env`
  3. Cân nhắc thêm cơ chế ép đổi mật khẩu lần đầu đăng nhập

### 4. Dependency Vulnerabilities (7 high severity — Server)
- **Packages:** `bcrypt@5.x` → `@mapbox/node-pre-gyp` → `tar`, `minimatch`, `qs`
- **Chi tiết:** 7 vulnerabilities (6 high, 1 low) trong dependency chain của `bcrypt`
- **Đời thường:** "Thư viện mã hóa mật khẩu đang dùng phiên bản cũ kéo theo nhiều lỗ hổng bảo mật. Hacker có thể khai thác qua path traversal hoặc ReDoS."
- **Cách sửa:**
  ```bash
  cd server
  npm audit fix          # Fix qs trước
  npm i bcrypt@6         # Breaking change: cần test lại
  # hoặc chuyển sang bcryptjs (pure JS, không cần native build)
  ```

## 🟡 Warnings (Nên sửa)

### 1. Token lưu trong `localStorage` — Dễ bị XSS
- **File:** `src/contexts/AuthContext.tsx` dòng 74, 86
- **Chi tiết:** JWT được lưu trong `localStorage`. Nếu xảy ra XSS, token bị đánh cắp ngay lập tức.
- **Đời thường:** "Nếu có một lỗi XSS nào đó, hacker lấy được token đăng nhập chỉ bằng 1 dòng `localStorage.getItem('token')`."
- **Cách sửa:** Chuyển sang `httpOnly` cookie (cần sửa cả server lẫn client), hoặc ít nhất thêm `Content-Security-Policy` header chặt.

### 2. CORS domain placeholder cho Production
- **File:** `server/index.ts` dòng 22
- **Chi tiết:** `origin: ['https://your-domain.com']` — Placeholder, chưa cấu hình domain thật.
- **Đời thường:** "Khi deploy lên production, CORS sẽ block toàn bộ request vì domain chưa được cấu hình đúng."
- **Cách sửa:** Thay bằng domain thực hoặc đọc từ biến môi trường `CORS_ORIGIN`.

### 3. Không validate format email/phone
- **File:** `server/routes/auth.ts` dòng 13-45
- **Chi tiết:** Khi đăng ký, `email` và `phone` được insert thẳng vào DB mà không kiểm tra format.
- **Đời thường:** "Người dùng có thể nhập `abc` vào ô email hoặc `xyz` vào ô điện thoại — vẫn đăng ký được."
- **Cách sửa:** Thêm regex validation cho email và phone trước khi insert.

### 4. AdminPage quá lớn (908 dòng)
- **File:** `src/pages/AdminPage.tsx`
- **Chi tiết:** Một component duy nhất chứa 26 functions, quản lý 5 tab (Dashboard, Users, Orders, Pricing, Deposit History).
- **Đời thường:** "File admin dài gần 1000 dòng, rất khó bảo trì. Mỗi lần sửa 1 tab có nguy cơ ảnh hưởng tab khác."
- **Cách sửa:** Tách thành sub-components: `AdminDashboard`, `AdminUsers`, `AdminOrders`, `AdminPricing`, `AdminDeposits`.

### 5. ServicePage lớn (670 dòng)
- **File:** `src/pages/ServicePage.tsx`
- **Chi tiết:** Component xử lý tất cả logic: form, submit, order history, UID conversion.
- **Cách sửa:** Tách `OrderHistoryTab` và form logic thành custom hooks.

### 6. Frontend dependency vulnerabilities (11 high — eslint/minimatch chain)
- **Chi tiết:** 11 vulnerabilities, hầu hết trong dev dependencies (eslint, typescript-eslint).
- **Đời thường:** "Các lỗ hổng này nằm ở công cụ phát triển, không ảnh hưởng production, nhưng nên cập nhật."
- **Cách sửa:** `cd I:\Website\Auto-like && npm audit fix --force`
